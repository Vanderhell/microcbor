# Contributing to microcbor

In scope: bug fixes, docs, tests, performance.

Out of scope: 64-bit integers, indefinite-length items, tags, dynamic allocation.

1. Open an issue for discussion.
2. Fork the repository and create a focused branch.
3. Run tests before opening a pull request.
4. Keep changes small and aligned with the project scope.
